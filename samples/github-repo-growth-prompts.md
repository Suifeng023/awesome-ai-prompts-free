# GitHub Repo Growth Prompts for AI Products

Use these free prompt templates to turn a quiet GitHub repository into a better discovery asset. They are designed for open-source samples, AI tools, prompt packs, templates, and micro-products.

## 1. README Above-the-Fold Audit

```text
You are a GitHub conversion copywriter.

Audit the first screen of this README for a repo that should attract [TARGET USER] and convert a fraction of readers to [NEXT ACTION].

Repo positioning: [ONE SENTENCE]
Current README opening:
[PASTE TOP 300 WORDS]

Return:
1. The clearest value proposition in one sentence
2. 5 problems in the current opening
3. A rewritten GitHub README opening with:
   - H1
   - one-line promise
   - 3 bullets showing outcomes
   - one proof/demo section
   - one soft CTA
4. A list of keywords to naturally include for GitHub + Google search
```

## 2. Free Sample Expansion Planner

```text
You are a product-led SEO strategist.

I have a GitHub repo with free samples for a paid digital product.

Paid product: [PRODUCT]
Target buyer: [BUYER]
Current free samples: [LIST]
Competitor keywords or topics: [LIST]

Suggest 10 new free sample files that are useful on their own but naturally lead to the paid product.

For each file, give:
- filename
- search-intent keyword
- 1-sentence promise
- 3 sections to include
- CTA angle that does not feel spammy

Prioritize files that a real user would bookmark or star.
```

## 3. Issue-Based Content Ideas

```text
You are a developer advocate.

Turn this repo into a useful community resource without pretending it has a large community yet.

Repo: [REPO DESCRIPTION]
Target users: [TARGET USERS]
Paid upgrade, if relevant: [PAID OFFER]

Create 8 GitHub issue ideas that could be opened by the maintainer as roadmap/content tasks.

Each issue should include:
- title
- why it matters
- acceptance criteria
- example content to add
- label suggestion

Avoid fake engagement. Make the issues genuinely useful as public roadmap items.
```

## 4. Product-Led Changelog Post

```text
You are writing a short build-in-public update for a technical audience.

Context:
- Repo/product: [NAME]
- What changed: [CHANGE]
- Why it matters: [USER BENEFIT]
- Link: [URL]

Write:
1. A GitHub release note under 150 words
2. A Dev.to / IndieHackers update under 250 words
3. A 5-post X/Twitter thread
4. A Hacker News Show HN title + body

Style: useful, specific, no hype, no exaggerated revenue claims.
```

## 5. Soft CTA Generator

```text
You are a conversion editor.

Write 12 soft CTAs for a free GitHub repo that links to a paid prompt pack.

Repo value: [FREE VALUE]
Paid product value: [PAID VALUE]
Audience: [AUDIENCE]
Tone: helpful, transparent, not pushy.

Group CTAs by placement:
- README top
- after a sample prompt
- end of guide
- GitHub release note

Each CTA must be one sentence and include a reason to click.
```
